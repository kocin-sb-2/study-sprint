Differentialekvationer — Differential Equations
▸
First-Order Differential Equations
Separable equations, linear first-order ODEs, integrating factors, initial value problems.
▸
Key Concepts
Separable Equations: Of the form dy/dx = f(x)g(y). Separate variables: g(y)⁻¹ dy = f(x) dx. Integrate both sides independently. Works when the equation can be written as a product of a function of x and a function of y.
Linear First-Order ODE: Standard form: dy/dx + P(x)y = Q(x). Solutions use integrating factor μ(x) = e^(∫P(x)dx). Multiply through: d/dx[μ(x)y] = μ(x)Q(x). Integrate to find y.
Initial Conditions: Specify a unique solution by giving y(x₀) = y₀. After solving the general solution with arbitrary constant C, apply the condition to find C explicitly. Essential for applied problems.
Integrating Factor Method: For dy/dx + P(x)y = Q(x), calculate μ(x) = e^(∫P(x)dx) carefully. The product rule then gives d/dx[μy] = μQ(x), which is always integrable.
Key Terms & Definitions
Ordinary Differential Equation (ODE): An equation involving an unknown function and its derivatives with respect to a single variable.
Order: The highest derivative in the equation (first-order = dy/dx; second-order = d²y/dx²).
General Solution: Contains arbitrary constant(s); represents a family of solutions.
Particular Solution: Obtained by applying initial/boundary conditions; specific curve in the family.
Homogeneous ODE: Right-hand side equals zero (e.g., dy/dx + P(x)y = 0).
Nonhomogeneous ODE: Right-hand side is nonzero (e.g., dy/dx + P(x)y = Q(x) with Q(x) ≠ 0).
Key Formulas
dy/dx = f(x)g(y) ⟹ ∫ dy/g(y) = ∫ f(x) dxSeparable equation separation.
dy/dx + P(x)y = Q(x) with μ(x) = e^(∫P(x)dx)Integrating factor for linear first-order.
d/dx[μ(x)y] = μ(x)Q(x)After multiplying by integrating factor, always integrable.
y = e^(-∫P(x)dx) [∫ Q(x)e^(∫P(x)dx) dx + C]General solution for linear first-order ODE.
Exam Focus
Solve a separable ODE and apply an initial condition to find a particular solution.
Identify whether an equation is separable or requires an integrating factor.
Calculate the integrating factor and solve a linear first-order ODE step-by-step.
Verify a solution by substitution: differentiate and check it satisfies the original equation.

Second-Order Linear ODEs with Constant Coefficients
Homogeneous and nonhomogeneous equations, characteristic equations, fundamental solutions.
▸
Key Concepts
Standard Form: d²y/dx² + a(dy/dx) + by = f(x), where a and b are constants. The complementary (homogeneous) solution y_c satisfies d²y/dx² + a(dy/dx) + by = 0. The particular solution y_p satisfies the full equation.
Characteristic Equation: Assume y = e^(rx) for homogeneous equation, yielding r² + ar + b = 0. Solve for roots r. The form of y_c depends on whether roots are real, complex, or repeated.
Real Distinct Roots: If r₁ ≠ r₂ are real, then y_c = C₁e^(r₁x) + C₂e^(r₂x). Two linearly independent exponential solutions.
Complex Roots: If r = α ± iβ (complex pair), then y_c = e^(αx)[C₁cos(βx) + C₂sin(βx)]. Oscillatory behavior with exponential envelope.
Repeated Real Root: If r₁ = r₂ = r (double root), then y_c = (C₁ + C₂x)e^(rx). The second solution includes a factor of x.
Key Terms & Definitions
Characteristic Equation: Polynomial equation r² + ar + b = 0 obtained by substituting y = e^(rx) into the homogeneous ODE.
Complementary Solution (y_c): General solution to the homogeneous equation d²y/dx² + a(dy/dx) + by = 0.
Particular Solution (y_p): Any specific solution to the nonhomogeneous equation.
General Solution: y = y_c + y_p; sum of complementary and particular solutions.
Linear Independence: Two solutions are independent if one is not a scalar multiple of the other; ensures two free constants in y_c.
Key Formulas
d²y/dx² + a(dy/dx) + by = 0 with characteristic equation r² + ar + b = 0Homogeneous second-order linear ODE.
Δ = a² - 4b determines root type: Δ > 0 (real distinct), Δ = 0 (repeated), Δ < 0 (complex)Discriminant analysis.
y_c = C₁e^(r₁x) + C₂e^(r₂x) (real distinct roots)General form with two exponential terms.
y_c = e^(αx)[C₁cos(βx) + C₂sin(βx)] (complex roots r = α ± iβ)Oscillatory solution with exponential envelope.
y_c = (C₁ + C₂x)e^(rx) (repeated root r)Second solution includes linear term.
Exam Focus
Form the characteristic equation from a second-order ODE and identify the root type (real distinct, repeated, or complex).
Write the complementary solution y_c based on root type.
Apply initial conditions y(x₀) and y'(x₀) to find C₁ and C₂.
Solve nonhomogeneous equations by finding y_p (via undetermined coefficients or variation of parameters) and adding y_c.

Applications of Differential Equations
Growth/decay models, harmonic motion, mixing problems, Newton's cooling law.
▸
Key Concepts
Exponential Growth/Decay: Model: dy/dt = ky with solution y(t) = y₀e^(kt). If k > 0: exponential growth (population, compound interest). If k < 0: exponential decay (radioactive decay, cooling). Half-life t₁/₂ = ln(2)/|k|.
Logistic Growth: Model: dy/dt = ky(L - y)/L, where L is carrying capacity. Solution approaches L asymptotically. More realistic for population dynamics than pure exponential growth due to resource constraints.
Newton's Law of Cooling: dT/dt = k(T - Tₐ), where T is object temperature, Tₐ is ambient temperature. Solution: T(t) = Tₐ + (T₀ - Tₐ)e^(kt). Describes cooling (or warming) of objects in isothermal environment.
Harmonic Motion: Undamped: d²x/dt² + ω²x = 0 with solution x(t) = A cos(ωt + φ). Damped: d²x/dt² + 2γ(dx/dt) + ω²x = 0. Underdamped oscillates with decaying amplitude; overdamped returns without oscillation.
Mixing Problems: Tank with inflow/outflow: dA/dt = (rate in) - (rate out). Concentration dynamics modeled by linear first-order ODE. Key: carefully track input/output rates and current concentration.
Key Terms & Definitions
Growth Constant k: Positive k yields exponential growth; negative k yields exponential decay.
Carrying Capacity L: Maximum sustainable population in logistic growth model.
Half-life: Time for quantity to reduce to half its initial value (e.g., radioactive decay).
Damping: Mechanism that reduces amplitude of oscillations over time (friction, air resistance).
Underdamped: Oscillatory behavior with decaying amplitude.
Overdamped: Non-oscillatory return to equilibrium; slower than critical damping.
Key Formulas
dy/dt = ky ⟹ y(t) = y₀e^(kt)Exponential growth/decay.
Half-life: t₁/₂ = ln(2)/|k|Time for quantity to halve in exponential decay.
dT/dt = k(T - Tₐ) ⟹ T(t) = Tₐ + (T₀ - Tₐ)e^(kt)Newton's law of cooling/heating.
d²x/dt² + 2γ(dx/dt) + ω²x = 0Damped harmonic oscillator equation.
dy/dt = ky(L - y)/LLogistic growth (solution y → L as t → ∞).
Exam Focus
Set up an exponential growth/decay model from a word problem and solve for unknown parameters using initial conditions.
Calculate half-life or doubling time from the growth constant.
Apply Newton's law of cooling: identify T₀, Tₐ, and k; solve for time to reach a target temperature.
Interpret damped oscillations: describe behavior and predict long-term motion.

Direction Fields and Qualitative Analysis
Slope fields, phase portraits, equilibrium points, stability analysis without solving explicitly.
▸
Key Concepts
Direction Field (Slope Field): For ODE dy/dx = f(x,y), plot small line segments with slope f(x,y) at grid points (x,y). Visualizes solution behavior without solving. Integral curves follow the field direction.
Equilibrium Points: For autonomous ODE dy/dx = f(y), equilibrium occurs where f(y) = 0. At equilibrium, y does not change (dy/dx = 0). Graphically, equilibrium is horizontal in the direction field.
Stability Analysis: At equilibrium y = y₀, check f'(y₀). If f'(y₀) < 0: stable (solutions converge). If f'(y₀) > 0: unstable (solutions diverge). If f'(y₀) = 0: higher-order analysis needed. Stable equilibria attract nearby solutions; unstable repel.
Phase Portrait: Plot equilibria and arrows indicating solution direction in a 1D phase line. For higher dimensions, plot trajectories in (x,y) plane showing evolution of solutions over time.
Key Terms & Definitions
Autonomous ODE: ODE of the form dy/dx = f(y) with no explicit x-dependence; behavior depends only on current y-value.
Equilibrium (Fixed Point): Value y₀ where f(y₀) = 0; solution remains constant y(x) = y₀.
Stable Equilibrium: Nearby solutions converge to it; corresponds to f'(y₀) < 0.
Unstable Equilibrium: Nearby solutions diverge; corresponds to f'(y₀) > 0.
Integral Curve: Solution curve in the (x,y) plane; everywhere tangent to direction field.
Key Formulas
dy/dx = f(x,y) defines slope at every point; plot small tangent segmentsDirection field construction.
For autonomous ODE dy/dx = f(y): equilibrium at f(y) = 0Equilibrium location.
Stability: check sign of f'(y₀) at equilibrium y₀Stability criterion (linearization).
Exam Focus
Sketch or interpret a direction field for a given ODE dy/dx = f(x,y).
Find equilibrium points of an autonomous ODE and determine their stability.
Draw a phase portrait for an autonomous ODE, showing equilibria and solution behavior.
Use qualitative analysis to predict long-term behavior without explicitly solving the ODE.

Existence and Uniqueness Theorems
Picard-Lindelöf theorem, well-posedness, when solutions exist and are unique.
▸
Key Concepts
Well-Posedness: An initial value problem is well-posed if: (1) a solution exists, (2) the solution is unique, (3) the solution depends continuously on initial conditions. Guarantees stability against perturbations.
Picard-Lindelöf Theorem: For initial value problem dy/dx = f(x,y), y(x₀) = y₀: if f and ∂f/∂y are continuous in a rectangular region containing (x₀,y₀), then a unique solution exists in a neighborhood of x₀. Failure of continuity can lead to non-unique or non-existent solutions.
Continuity Requirement: Discontinuous f or ∂f/∂y can cause branching (multiple solutions) or gaps (no solution). Example: dy/dx = √y with y(0) = 0 has non-unique solutions y = 0 and y = x²/4.
Existence Without Uniqueness: Some equations have solutions that exist but are not unique due to non-Lipschitz behavior. Geometrically, integral curves may cross.
Key Terms & Definitions
Lipschitz Condition: f satisfies a Lipschitz condition if |f(x,y₁) - f(x,y₂)| ≤ L|y₁ - y₂| for some constant L; ensures uniqueness.
Continuity: f is continuous on a region if small changes in (x,y) yield small changes in f(x,y).
Initial Value Problem (IVP): ODE with specified initial condition y(x₀) = y₀.
Maximal Interval of Existence: Largest interval on which a solution is guaranteed to exist.
Key Formulas
Picard-Lindelöf: If f and ∂f/∂y continuous in rectangle R containing (x₀,y₀), then unique solution existsSufficient condition for well-posedness.
Exam Focus
Determine whether a given IVP satisfies Picard-Lindelöf conditions; justify existence and uniqueness.
Identify situations where solutions are non-unique (e.g., dy/dx = √y) and explain the failure of continuity.

Komplexa Tal — Complex Numbers
▸
Complex Arithmetic and Algebraic Properties
Definition, addition, subtraction, multiplication, division, complex conjugate, modulus.
▸
Key Concepts
Definition and Form: Complex number z = a + bi where a (real part) and b (imaginary part) are real numbers, i² = -1. Denote Re(z) = a, Im(z) = b. The set ℂ of complex numbers extends ℝ (real numbers).
Addition and Subtraction: (a + bi) + (c + di) = (a+c) + (b+d)i. (a + bi) - (c + di) = (a-c) + (b-d)i. Performed component-wise on real and imaginary parts.
Multiplication: (a + bi)(c + di) = ac - bd + (ad + bc)i. Use FOIL and i² = -1. Commutative and associative; distributive over addition.
Complex Conjugate: For z = a + bi, conjugate z* = a - bi. Properties: (z + w)* = z* + w*, (zw)* = z*w*, zz* = |z|². Used in division: 1/z = z*/|z|².
Modulus (Absolute Value): |z| = √(a² + b²) for z = a + bi. Geometric interpretation: distance from origin to point (a,b) in complex plane. Properties: |zw| = |z||w|, |z/w| = |z|/|w|, triangle inequality |z+w| ≤ |z| + |w|.
Key Terms & Definitions
Real Part: Re(z) = a for z = a + bi.
Imaginary Part: Im(z) = b for z = a + bi (note: Im(z) is real, not imaginary).
Purely Imaginary: Complex number with real part zero (z = bi).
Purely Real: Complex number with imaginary part zero (z = a).
Complex Conjugate: z* obtained by negating the imaginary part; reflects z across the real axis.
Modulus: |z| = √(a² + b²); distance from origin to z in the complex plane.
Key Formulas
(a + bi) + (c + di) = (a+c) + (b+d)iComplex addition.
(a + bi)(c + di) = (ac - bd) + (ad + bc)iComplex multiplication.
1/(a + bi) = (a - bi)/(a² + b²)Complex division; multiply by conjugate.
|z| = √(a² + b²)Modulus of z = a + bi.
zz* = |z|²Product of z and its conjugate.
Exam Focus
Perform arithmetic operations: add, subtract, multiply, or divide complex numbers; simplify results.
Rationalize complex denominators using conjugates.
Calculate modulus and identify real/imaginary parts.
Apply properties: |z₁z₂| = |z₁||z₂|, (zw)* = z*w*.

Complex Plane and Geometric Interpretation
Argand diagram, polar coordinates, geometric operations, distance and angles.
▸
Key Concepts
Argand Diagram: 2D plane with real axis (horizontal) and imaginary axis (vertical). Each complex number z = a + bi plots as point (a, b). Addition is vector addition; multiplication involves rotation and scaling.
Modulus as Distance: |z| = distance from origin (0, 0) to point z. |z - w| = distance between points z and w in the complex plane. Circle |z| = r is centered at origin with radius r.
Argument (Angle): arg(z) = angle θ such that z = r(cos θ + i sin θ), where r = |z| and θ ∈ [0, 2π) or (-π, π]. tan θ = b/a (careful with quadrant). arg(z₁z₂) = arg(z₁) + arg(z₂) (modulo 2π).
Geometric Operations: Addition: parallelogram rule. Subtraction: vector from z₂ to z₁ is z₁ - z₂. Multiplication by i: rotates counterclockwise by 90°. Multiplication by r > 0: scaling by factor r.
Key Terms & Definitions
Argument: Angle θ from positive real axis to line from origin to z; arg(z) ∈ [0, 2π) or arg(z) ∈ (-π, π].
Principal Argument: Usually the unique value in (-π, π] or [0, 2π).
Geometric Multiplication: Multiply moduli, add arguments: |z₁z₂| = |z₁||z₂|, arg(z₁z₂) = arg(z₁) + arg(z₂).
Rotation: Multiplication by e^(iθ) rotates counterclockwise by angle θ.
Key Formulas
|z| = √(a² + b²)Distance from origin.
arg(z) = arctan(b/a) (adjust for quadrant)Angle from real axis; use atan2(b,a) for correct quadrant.
|z₁ - z₂| = distance between z₁ and z₂Distance formula in complex plane.
|z₁z₂| = |z₁||z₂| and arg(z₁z₂) = arg(z₁) + arg(z₂)Multiplication in Argand diagram.
Exam Focus
Plot complex numbers on an Argand diagram; find modulus and argument.
Use geometric operations: visualize multiplication, division, rotation in the complex plane.
Solve geometric problems using complex numbers (e.g., finding vertices of polygons, angles).

Polar Form and Euler's Formula
r·e^(iθ) form, exponential representation, conversions, trigonometric connections.
▸
Key Concepts
Polar Form: Complex number z = r(cos θ + i sin θ), where r = |z| ≥ 0 (modulus) and θ = arg(z) (argument). Compact notation: z = r∠θ or z = r cis θ (where cis = cos + i sin).
Euler's Formula: e^(iθ) = cos θ + i sin θ. This profound identity links exponentials and trigonometry. Allows writing z = re^(iθ) in exponential form.
Conversion to Polar: Given z = a + bi: r = √(a² + b²), θ = arctan(b/a) (adjust for quadrant). Reverse: a = r cos θ, b = r sin θ.
Operations in Polar Form: Multiplication: (r₁e^(iθ₁))(r₂e^(iθ₂)) = r₁r₂e^(i(θ₁+θ₂)). Division: (r₁e^(iθ₁))/(r₂e^(iθ₂)) = (r₁/r₂)e^(i(θ₁-θ₂)). Exponentiation via De Moivre (next section).
Key Terms & Definitions
Polar Coordinates (r, θ): r is modulus, θ is argument; represent a point in the complex plane.
Exponential Form z = re^(iθ): Compact notation for complex number using Euler's formula.
cis(θ): Shorthand for cos θ + i sin θ; equals e^(iθ).
Conversion: From rectangular (a + bi) to polar (r, θ) and vice versa.
Key Formulas
e^(iθ) = cos θ + i sin θEuler's formula; fundamental connection between exponentials and trigonometry.
z = r e^(iθ) = r(cos θ + i sin θ)Polar (exponential) form with r = |z|, θ = arg(z).
z₁z₂ = r₁r₂ e^(i(θ₁ + θ₂))Multiplication in polar form: moduli multiply, arguments add.
z₁/z₂ = (r₁/r₂) e^(i(θ₁ - θ₂))Division in polar form.
r = √(a² + b²), θ = arctan(b/a) (with quadrant adjustment)Conversion from rectangular to polar.
Exam Focus
Convert between rectangular (a + bi) and polar (r e^(iθ)) forms.
Multiply or divide complex numbers using polar form; verify moduli and arguments.

De Moivre's Theorem and Powers
Computing nth powers, nth roots, applications to trigonometric identities.
▸
Key Concepts
De Moivre's Theorem: For z = r(cos θ + i sin θ) and integer n: z^n = r^n(cos(nθ) + i sin(nθ)). In exponential form: (re^(iθ))^n = r^n e^(inθ). Powerful for computing powers without expanding binomially.
nth Roots: The nth roots of z = re^(iθ) are w_k = r^(1/n) e^(i(θ + 2πk)/n) for k = 0, 1, ..., n-1. There are exactly n distinct nth roots, evenly spaced on a circle of radius r^(1/n), separated by angle 2π/n.
Roots of Unity: The nth roots of 1 are e^(2πik/n) for k = 0, 1, ..., n-1. Denoted as ω_n^k where ω_n = e^(2πi/n). Geometrically, they form a regular n-gon on the unit circle, centered at origin.
Trigonometric Identities: De Moivre's theorem connects powers of (cos θ + i sin θ) to multiple-angle formulas. E.g., (cos θ + i sin θ)^3 = cos 3θ + i sin 3θ expands to cos³θ - 3cos θ sin²θ + i(...), yielding cos 3θ = cos³θ - 3cos θ sin²θ.
Key Terms & Definitions
De Moivre's Theorem: [r(cos θ + i sin θ)]^n = r^n(cos(nθ) + i sin(nθ)).
nth Root: A complex number w such that w^n = z.
Root of Unity: Complex number z such that z^n = 1; nth roots of unity form regular n-gon on unit circle.
Primitive nth Root of Unity: e^(2πi/n) = ω_n; all nth roots of unity are powers of ω_n.
Key Formulas
z^n = r^n(cos(nθ) + i sin(nθ)) or z^n = r^n e^(inθ)De Moivre's theorem for powers.
w_k = r^(1/n) e^(i(θ + 2πk)/n) for k = 0, 1, ..., n-1All nth roots of z = r e^(iθ).
e^(2πik/n) for k = 0, 1, ..., n-1nth roots of unity; sum = 0 for n > 1.
Exam Focus
Use De Moivre's theorem to compute high powers of complex numbers efficiently.
Find all nth roots of a complex number; identify the principal root and express others.
Solve equations like z^n = a by converting to polar and finding roots.
Derive trigonometric identities using De Moivre (e.g., cos(2θ), cos(3θ)).

Polynomial Equations and Complex Solutions
Fundamental theorem of algebra, factor theorem, complex conjugate roots, solving cubics and quartics.
▸
Key Concepts
Fundamental Theorem of Algebra: Every polynomial p(z) of degree n ≥ 1 with complex (or real) coefficients has at least one complex root. Equivalently, p(z) factors as p(z) = a(z - z₁)(z - z₂)...(z - z_n) for some complex numbers z₁, ..., z_n.
Complex Conjugate Root Theorem: If p(z) has real coefficients and z₀ = a + bi is a root, then z₀* = a - bi is also a root. Allows factoring real polynomials into real quadratics (irreducible over ℝ).
Factor Theorem: z₀ is a root of p(z) if and only if (z - z₀) is a factor: p(z) = (z - z₀)q(z) for some polynomial q(z). Degree of q is n - 1.
Polynomial Factorization: Over ℂ: every polynomial factors completely into linear factors. Over ℝ: factors are linear (from real roots) or irreducible quadratics (from complex conjugate pairs). Find roots via rational root theorem, synthetic division, or formulas (quadratic, cubic, etc.).
Key Terms & Definitions
Root (Zero): Value z₀ where p(z₀) = 0.
Multiplicity: If (z - z₀)^m divides p(z) but (z - z₀)^(m+1) does not, then z₀ has multiplicity m.
Complex Conjugate Root Theorem: For polynomials with real coefficients, non-real roots come in conjugate pairs.
Irreducible Quadratic: Real quadratic with no real roots (e.g., x² + 1); factors into linear terms over ℂ.
Key Formulas
p(z) = a_n z^n + a_(n-1) z^(n-1) + ... + a_1 z + a_0General polynomial of degree n.
p(z) = a(z - z₁)(z - z₂)...(z - z_n)Complete factorization over ℂ (fundamental theorem of algebra).
For real p: complex roots occur in conjugate pairsComplex conjugate root theorem.
z = [-b ± √(b² - 4ac)] / (2a)Quadratic formula (works over ℂ).

Combinatorics: Permutations and Combinations
Counting principles, permutations, combinations, binomial coefficients, Pascal's triangle.
▸
Key Concepts
Multiplication Principle: If event A has m outcomes and event B has n outcomes, total outcomes = m × n. Extends: (m₁)(m₂)...(m_k) for k events. Fundamental for counting composite outcomes.
Permutations: Ordered arrangements. P(n,r) = n!/(n-r)! counts arrangements of r objects from n distinct objects. P(n,n) = n! (all n objects). Account for order: AB ≠ BA.
Combinations: Unordered selections. C(n,r) = n!/[r!(n-r)!] counts subsets of r objects from n distinct objects. Order doesn't matter: {A,B} = {B,A}. Also written as (n choose r).
Binomial Theorem: (x + y)^n = Σ C(n,k) x^(n-k) y^k for k from 0 to n. Coefficients are binomial coefficients C(n,k). Useful for expanding powers and combinatorial proofs.
Pascal's Triangle: Triangular array where each entry is sum of two above: C(n,k) = C(n-1,k-1) + C(n-1,k). Row n contains coefficients of (x+y)^n. Pattern: symmetry, row sums = 2^n.
Key Terms & Definitions
Permutation: Ordered arrangement of objects; order matters.
Combination: Unordered selection of objects; order doesn't matter.
Factorial n!: n! = n × (n-1) × ... × 2 × 1; also 0! = 1.
Binomial Coefficient C(n,k): Number of k-element subsets of an n-element set.
Key Formulas
P(n,r) = n!/(n-r)!Permutations of r from n; order matters.
C(n,r) = n!/[r!(n-r)!] = P(n,r)/r!Combinations of r from n; order doesn't matter.
(x + y)^n = Σ_{k=0}^n C(n,k) x^(n-k) y^kBinomial theorem; sum of binomial terms.
C(n,k) = C(n-1,k-1) + C(n-1,k)Pascal's triangle recurrence.
Σ_{k=0}^n C(n,k) = 2^nSum of row n in Pascal's triangle.
Exam Focus

Number Theory Basics
Divisibility, prime numbers, GCD, modular arithmetic, Euclidean algorithm.
▸
Key Concepts
Divisibility: Integer a divides b (a | b) if b = ak for some integer k. Also: a is a divisor of b; b is a multiple of a. Properties: reflexive (a | a), transitive (a | b and b | c ⟹ a | c), linear combination (a | b and a | c ⟹ a | (bx + cy) for any integers x, y).
Prime Numbers: Integer p > 1 with exactly two positive divisors (1 and p). Fundamental theorem of arithmetic: every integer > 1 factors uniquely (up to order) as product of primes. Used in cryptography and number theory.
Greatest Common Divisor (GCD): gcd(a,b) is the largest positive integer dividing both a and b. Euclidean algorithm computes gcd efficiently via repeated division. Linear Bézout relation: gcd(a,b) = ax + by for some integers x, y.
Modular Arithmetic: a ≡ b (mod n) means n | (a - b). Equivalence relation. Operations: (a ± b) mod n = [(a mod n) ± (b mod n)] mod n. Multiplicative inverse: a⁻¹ (mod n) exists iff gcd(a,n) = 1, found via extended Euclidean algorithm.
Key Terms & Definitions
Divisible: a is divisible by b if b | a (remainder is 0).
Prime: Natural number > 1 with no divisors except 1 and itself.
Composite: Natural number > 1 that is not prime; has prime factorization.
GCD (Greatest Common Divisor): Largest positive integer dividing both numbers.
Coprime (Relatively Prime): gcd(a,b) = 1; a and b share no common prime factors.
Congruence a ≡ b (mod n): a and b have the same remainder when divided by n.
Key Formulas
Euclidean Algorithm: gcd(a,b) = gcd(b, a mod b) (repeat until remainder is 0)Efficient computation of GCD.
Fundamental Theorem of Arithmetic: n = p₁^(e₁) × p₂^(e₂) × ... × p_k^(e_k)Unique prime factorization.
gcd(a,b) = ax + by (Bézout's identity)Linear combination; x, y found via extended Euclidean algorithm.
a ≡ b (mod n) ⟺ n | (a - b)Definition of modular congruence.

Mathematical Induction
Principle of induction, strong induction, well-ordering, proof structure and verification.
▸
Key Concepts
Principle of Mathematical Induction: To prove P(n) for all n ≥ n₀: (1) Base case: verify P(n₀) is true. (2) Inductive step: assume P(k) (inductive hypothesis), prove P(k+1). If both hold, P(n) is true for all n ≥ n₀. Works because the integers have no descending chains.
Strong Induction: Variant where inductive hypothesis assumes P(i) for all n₀ ≤ i ≤ k, then proves P(k+1). Stronger (easier to use) than weak induction when later terms depend on multiple predecessors (e.g., Fibonacci, recurrence relations).
Well-Ordering Principle: Every nonempty set of positive integers has a minimum element. Equivalent to mathematical induction. Often used for existence proofs: assume a counterexample exists, derive contradiction from minimality.
Common Applications: Summation formulas (Σk = n(n+1)/2), divisibility properties, inequalities (Bernoulli's inequality), combinatorial identities, recurrence relations.
Key Terms & Definitions
Base Case: Verification that P(n₀) is true (usually n₀ = 0 or 1).
Inductive Hypothesis: Assumption that P(k) is true for some k ≥ n₀.
Inductive Step: Proof that P(k) ⟹ P(k+1).
Strong Induction: Inductive hypothesis assumes P(i) for all i ≤ k, then proves P(k+1).
Key Formulas
Σ_{k=1}^n k = n(n+1)/2Sum of first n integers (proven by induction).
Σ_{k=1}^n k² = n(n+1)(2n+1)/6Sum of squares (proven by induction).
Σ_{k=0}^n r^k = (r^(n+1) - 1)/(r - 1) for r ≠ 1Geometric series (proven by induction).
Exam Focus

Use strong induction when base case or inductive step requires multiple previous values.

Graph Theory Basics
Vertices, edges, paths, cycles, Euler and Hamilton paths, trees, connectivity.
▸
Key Concepts
Graph Definition: G = (V, E) where V is set of vertices and E is set of edges (pairs of vertices). Undirected: edges are unordered pairs {u,v}. Directed: edges are ordered pairs (u,v). Weighted graph assigns values to edges. Multigraph allows multiple edges between vertices.
Paths and Cycles: Walk: sequence of vertices with consecutive edges. Path: walk with no repeated vertices. Cycle: walk starting/ending at same vertex, no other repeats. Connected graph: path between every pair of vertices. Acyclic graph: contains no cycles (like trees).
Euler Path/Circuit: Path using every edge exactly once. Euler circuit: closed path using every edge once. Exists iff (undirected) connected and all vertices have even degree, or exactly two vertices have odd degree (path exists then). Directed: in-degree = out-degree for circuit.
Hamilton Path/Cycle: Path visiting every vertex exactly once. Hamilton cycle: closed path visiting every vertex once. No simple criterion (unlike Euler); NP-hard to determine in general. Useful in routing and scheduling problems.
Trees: Connected acyclic graph. n vertices, n-1 edges. Unique path between any two vertices. Spanning tree: subgraph connecting all vertices with no cycles. Tree properties: minimal connected, maximal acyclic.
Key Terms & Definitions
Vertex (Node): Fundamental unit in a graph.
Edge: Connection between two vertices.
Degree: Number of edges incident to a vertex (in-degree and out-degree for directed graphs).
Connected: Graph where path exists between every pair of vertices.
Tree: Connected acyclic graph; n vertices have n-1 edges.
Spanning Tree: Subgraph including all vertices and n-1 edges, forming a tree.

Tree with n vertices has n-1 edgesFundamental property of trees.
Euler path exists ⟺ connected and (0 or 2 vertices have odd degree)Necessary and sufficient condition for Euler path.

Sequences, Series, and Recursion
Arithmetic and geometric sequences, convergence, telescoping series, recurrence relations.
▸
Key Concepts
Arithmetic Sequence: a_n = a₁ + (n-1)d where d is common difference. Sum: S_n = n(a₁ + a_n)/2 = n[2a₁ + (n-1)d]/2. Linear growth pattern; difference between consecutive terms is constant.
Geometric Sequence: a_n = a₁ r^(n-1) where r is common ratio. Sum: S_n = a₁(1 - r^n)/(1 - r) for r ≠ 1. Exponential growth/decay. Infinite series: Σ a₁ r^(n-1) = a₁/(1-r) if |r| < 1 (convergent).
Convergence: Sequence {a_n} converges to L if for any ε > 0, |a_n - L| < ε for all sufficiently large n. Geometric sequence converges to 0 if |r| < 1. Arithmetic sequence diverges unless d = 0.
Recurrence Relations: Equation relating a_n to previous terms (a_(n-1), a_(n-2), etc.). Example: Fibonacci a_n = a_(n-1) + a_(n-2). Solve via characteristic equation (for linear constant-coefficient), iteration, or closed-form formula. Proved by induction.
Telescoping Series: Series where consecutive terms partially cancel. Σ (a_n - a_(n+1)) = a₁ - lim a_n. Useful for evaluating sums of rational expressions with partial fractions.
Key Terms & Definitions
Common Difference d: Constant difference between consecutive terms in arithmetic sequence.
Common Ratio r: Constant ratio between consecutive terms in geometric sequence.
Convergent Series: Infinite series with finite sum (partial sums approach limit).
Divergent Series: Infinite series whose partial sums don't approach a finite limit.
Recurrence Relation: Equation defining a_n in terms of previous terms.
Key Formulas
Arithmetic: a_n = a₁ + (n-1)d, S_n = n(a₁ + a_n)/2Arithmetic sequence and sum.
Geometric: a_n = a₁ r^(n-1), S_n = a₁(1 - r^n)/(1 - r)Geometric sequence and sum (r ≠ 1).
Geometric series: Σ_{n=0}^∞ r^n = 1/(1-r) if |r| < 1Infinite geometric series (convergent).
Fibonacci: a_n = a_(n-1) + a_(n-2), closed form a_n = (φ^n - ψ^n)/√5Famous recurrence; φ = (1+√5)/2, ψ = (1-√5)/2.

Proof Techniques and Logical Reasoning
Direct proof, proof by contradiction, proof by contrapositive, logical connectives and quantifiers.
▸
Key Concepts
Direct Proof: Assume hypothesis P; derive conclusion Q through logical steps using definitions, known theorems, and valid inference. Most common and straightforward method. Example: "If n is even, then n² is even. Proof: n = 2k, so n² = 4k² = 2(2k²), which is even."
Proof by Contradiction: Assume P and NOT Q (negation of conclusion). Derive a logical contradiction (e.g., false statement or P and NOT P). Conclude Q must follow from P. Effective when direct proof is hard. Example: Proving √2 is irrational by assuming it's rational and deriving m, n both even (contradiction).
Proof by Contrapositive: To prove P ⟹ Q, prove NOT Q ⟹ NOT P (logically equivalent). Useful when contrapositive is simpler. Example: "If n² is even, then n is even" proven via contrapositive: "If n is odd, then n² is odd."
Logical Operators: AND (∧, both true), OR (∨, at least one true), NOT (¬, negation), IMPLIES (⟹, conditional), IFF (⟺, if and only if, biconditional). De Morgan's laws: ¬(P ∧ Q) = (¬P) ∨ (¬Q), ¬(P ∨ Q) = (¬P) ∧ (¬Q).
Quantifiers: Universal (∀: "for all"), Existential (∃: "there exists"). Negating: ¬(∀x P(x)) = ∃x ¬P(x), ¬(∃x P(x)) = ∀x ¬P(x). Essential for precise mathematical statements.
Key Terms & Definitions
Hypothesis (Premise): Assumed statement P in a conditional P ⟹ Q.
Conclusion (Consequent): Statement Q to be proven in P ⟹ Q.
Contrapositive of P ⟹ Q: ¬Q ⟹ ¬P; logically equivalent to original.
Biconditional P ⟺ Q: (P ⟹ Q) ∧ (Q ⟹ P); "if and only if"; P and Q are equivalent.
Tautology: Statement always true (e.g., P ∨ ¬P).
Contradiction: Statement always false (e.g., P ∧ ¬P).
Key Formulas
De Morgan's Laws: ¬(P ∧ Q) = (¬P) ∨ (¬Q), ¬(P ∨ Q) = (¬P) ∧ (¬Q)Negation of compound propositions.
Contrapositive: (P ⟹ Q) ≡ (¬Q ⟹ ¬P)Logical equivalence; same truth value.
Biconditional: (P ⟺ Q) = (P ⟹ Q) ∧ (Q ⟹ P)Two-way implication.
Exam Focus
Write a direct proof: clearly state assumptions and logical steps.
Prove a statement using contrapositive when the contrapositive is simpler.
Use proof by contradiction: assume the negation and derive an absurdity.
Negate quantified statements correctly; identify logical structure (AND, OR, IMPLIES).

Advanced Problem-Solving Strategies
Working backwards, symmetry, invariants, pigeonhole principle, extreme cases.
▸
Key Concepts
Working Backwards: Start from desired conclusion and reverse-engineer conditions needed. Identify what intermediate results would suffice. Especially useful for olympiad problems and optimization. Example: To maximize f(x), identify critical points by setting f'(x) = 0, then evaluate candidates.
Symmetry: Recognize symmetric structure to simplify. Symmetric functions (f(x,y) = f(y,x)), symmetric configurations reduce cases. Example: By symmetry, max/min often occur at symmetric points (e.g., maximum of f(x,y) on unit circle at x = y).
Invariants: Quantity or property unchanged by transformations. Find invariant to establish constraints or impossibilities. Example: Checkerboard coloring invariant shows certain puzzles (rook moves, tiling) are impossible.
Pigeonhole Principle: If n items into m containers with n > m, at least one container has ≥ 2 items. Generalizes: ⌈n/m⌉ items in some container. Useful for existence proofs. Example: Among 13 people, at least two share a birth month.
Extreme Cases: Test edge/boundary values, very small/large values, degenerate cases. Constraints on extrema often reveal structure. Example: In optimization, check boundary and critical interior points.
Key Terms & Definitions
Invariant: Quantity or property preserved under specified operations/transformations.
Symmetric Function: Function unchanged by certain permutations of variables.
Pigeonhole Principle (Dirichlet): With n > m items and m containers, some container has ≥ 2 items.
Degenerate Case: Special/limiting case, often boundary conditions or zero values.
